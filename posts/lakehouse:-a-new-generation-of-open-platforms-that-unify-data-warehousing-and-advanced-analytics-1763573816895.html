The use of open formats helped in implementing machine learning.
From 2015 cloud data lakes like s3, ADLS, GCS started replacing HDFS{ superior durability, geo replication, extremely low cost with possibility of automatic, even cheaper, archival storage (eg: AWS Glacier).

Operation Database-->ETL-->DW-->BI ( First Generation analytics)

Raw Source-->ETL-->Data lake----> ETL---> DW(red shift/ snowflake)-->Bi [ Two -tier architecture data lake+ data warehouse].
mostly used by fortune 500 companies.

The above architecture suffers from 4 problems:
  • Reliability: Keeping data consistent in data lake and DW is a difficult and costly, we have continuously engineer ETL which may introduce bugs that reduce data quality.
  • Data Staleness: The data in the warehouse is stale compared to that of data lake, with new data frequently taking days to load, a drawback to this when compared with first generation.
  • Limited support for advanced analytics: Data warehouse cannot support the property of using machine learning as it is not compatible to run on top of it, which introduced another ETL step to use them as files for further predictions. If we use the data lake then we may loose the property of ACID transactions, data versioning and indexing.
  • Total Cost of ownership: Apart from ETL, users pay double the storage cost for data copied into a warehouse, which stores data in propietary formats that increase the cost of migrating data or workloads to other systems.
