Optimization :

Data file layout means organization and storage of files that make up the delta tables.

Optimizing layouts help in data skipping algorithm

Techniques include:
  • Partitioning { organizing table by grouping rows that share same value for predefined partition columns}
create table emp_part(columns)
partitioned by(column_name)
  • Z order by{ grouping similar data into optimized files without creating directories/data skipping is used very well/ high cardinality column can be benefited}
optimize table_name
zorder by column
  • Liquid clustering{ Improved version of z order by / can be altered in between }
create table table_name 
( columns
)
cluster by (clustering_columns)
or 
alter table table_name
cluster by (new-columns)
  • Automatic liquid clustering { Databricks chooses the clustering key depending on the query requests: Requires predictive optimization on unity catalog managed tables}
create table table_name(columns)
cluster by auto
or
alter table table_name
cluster by auto
